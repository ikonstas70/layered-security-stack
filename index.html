⚙️Stack Components

Each layer has a specific responsibility. Defense in depth — no single point of failure.

🔐

WireGuard VPN

Full-tunnel client/server. All traffic — IPv4 and IPv6 — encrypted and routed through the VPS. Client real IP never reaches the destination.

ChaCha20-Poly1305 · UDP · AllowedIPs 0/0
🌐

dnscrypt-proxy

DNS-over-HTTPS server running on the VPS, served only to VPN clients. All DNS queries are encrypted inside the tunnel before leaving the VPS.

DoH · 1.1.1.1 · 9.9.9.9 · no plain-text DNS
👁

Snort IDS

Passive packet inspection on WAN, VPN, and container interfaces. Custom rules detect SSH brute force, web shells, SQLi, path traversal, and recon.

SID 9000001–9000008 · fast alert mode
📊

Wazuh SIEM

Centralized log and event collection. Agent on client ships FIM alerts, vulnerability data, and auth events to the manager for correlation and alerting.

FIM · Vuln detection · Active response
🏠

OSSEC

Host-based IDS running alongside Wazuh. Monitors system files, auth logs, and processes. Active response can ban IPs or kill processes on detection.

HIDS · log analysis · file integrity
🧱

iptables + ipset

NAT masquerade for VPN forwarding. ipset blocklists updated daily from Feodo, Emerging Threats, and Tor exit feeds. Zero-downtime atomic swaps.

ipset hash:ip · 1,800+ blocked IPs
🚫

fail2ban

SSH brute-force protection. Monitors auth logs and inserts iptables DROP rules after N failed attempts. Automatically expires bans.

sshd jail · iptables · auto-expire
☁️

Cloudflare Tunnel

Exposes internal web services without opening any inbound ports. Traffic arrives encrypted from Cloudflare's edge — no public IP exposure of origin.

QUIC/H2 · zero open ports · DDoS shielded

🏗Key Design Decisions

Non-obvious choices and the reasoning behind them.

WireGuard: INSERT not APPEND to FORWARD chain

Docker adds a REJECT rule to the iptables FORWARD chain. Using -A (append) places your WireGuard ACCEPT rule after Docker's REJECT — VPN traffic is silently dropped.

# Wrong — sits after Docker's REJECT
iptables -A FORWARD -i wg0 -j ACCEPT

# Correct — inserted at position 1, before Docker
iptables -I FORWARD 1 -i wg0 -j ACCEPT

IPv6 ULA prefix prevents ISP address leaks

A full tunnel with only AllowedIPs = 0.0.0.0/0 still leaves IPv6 exposed. Assign a ULA prefix (fd00::/8) to the VPN interface and include ::/0 in AllowedIPs.

Multiple tunnels: specific host route wins over subnet route

When two WireGuard tunnels claim overlapping subnets, the secondary tunnel's route blackholes traffic to the VPN server's internal IP.

IFACE=$(route -n get 8.8.8.8 | awk '/interface:/{print $2}')
route add -host VPN_INTERNAL_IP -interface "$IFACE"

TLS: enforce 1.2/1.3, remove legacy from default config

nginx ships a global ssl_protocols directive that includes TLS 1.0/1.1. Remove it at the global level.

ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers "ECDHE-ECDSA-AES128-GCM-SHA256:...";
ssl_prefer_server_ciphers off;

SSH: always maintain a tunnel-based alias

Cloud firewall rules may block port 22 from unknown source IPs. The VPN server's internal IP is only reachable through the tunnel — bypassing external firewall rules entirely.

🔧Operational Fixes

Real issues encountered in production and how they were resolved.

RESOLVED

Full tunnel + unreachable VPN server = total internet loss

AllowedIPs 0.0.0.0/0 routes all traffic into the tunnel. If the VPN server goes down, all traffic disappears. Fix: bring down the tunnel to restore the default gateway.

sudo wg-quick down /path/to/client.conf
RESOLVED

Two active tunnels blackhole traffic to VPN server internal IP

Secondary tunnel claims the VPN subnet. SSH and DNS to the VPN server stop working. Fix: add a specific /32 host route through the primary tunnel interface.

RESOLVED

DNS leaking to public fallback due to routing conflict

Same routing bug causes DNS queries to VPN server to be blackholed. OS falls back to secondary public DNS — queries are unencrypted and outside the tunnel.

RESOLVED

nginx serving TLS 1.0/1.1 — SSLv2 exploit attempt detected

Snort detected an SSLv2 Client_Hello overflow attempt. Removed legacy protocols from global nginx config, enforced TLS 1.2/1.3 only, reloaded zero-downtime.
